# Security Policy

## Reporting

If you find a security issue, do not open a public GitHub issue.

Report it privately to `jakob@jait.dev` with:

- a short description of the issue
- impact and affected components
- reproduction steps or proof of concept
- any suggested mitigation

## Response

I will acknowledge receipt, validate the report, and work on a fix as quickly as possible. Please give reasonable time for remediation before public disclosure.

## Scope

This project includes local-first tooling with terminal, filesystem, browser, and network capabilities. Reports involving privilege boundaries, path escapes, SSRF, consent bypasses, auth flaws, or secret exposure are especially useful.
